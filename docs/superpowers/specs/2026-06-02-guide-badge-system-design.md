# Guide & Badge System — Design Spec
**Date:** 2026-06-02  
**Status:** Approved

---

## 배경 및 목표

교수님 피드백: "커뮤니티에서 정보를 얻을 수 있다면 가이드가 굳이 필요한가?"

**핵심 답변:** 가이드는 커뮤니티 신뢰도 시스템의 기반이다.

```
가이드(학습) → 퀴즈(검증) → 뱃지(증명) → 커뮤니티(활용)
```

커뮤니티 QnA에서 "수강신청 박사" 뱃지를 가진 유저의 답변은 신뢰도가 다르다.  
가이드 없이는 뱃지가 없고, 뱃지 없이는 커뮤니티 신뢰도가 없다.

**타겟:** 신입 유학생 (메인) / 재학생 (세컨더리)

---

## 전체 흐름

```
[가이드 화면]
  └─ 카테고리 카드 (노션 콘텐츠 기반)
       ├─ 읽기 (카드 상세)
       └─ 퀴즈 도전 버튼
            └─ 카테고리 퀴즈 (객관식 4지선다)
                 └─ 70% 이상 통과
                      └─ 뱃지 획득
                           ├─ 프로필 도감에 표시
                           └─ 커뮤니티 게시글/답변 옆 표시 (팀원 담당)
```

## 섹션 1: 가이드 콘텐츠

### 변경 내용
- 현재 `src/data/khuGuide.ts`의 하드코딩 데이터를 노션 텍스트로 전면 교체
- 카테고리 구성은 노션 파일 기준으로 확정
- UI 구조(카드 그리드, 상세 뷰, 한/영 토글)는 유지

### 각 카테고리 카드 변경
- 현재: 팁 개수 표시
- 변경: 뱃지 획득 여부 표시 (획득 시 뱃지 아이콘 + 색상, 미획득 시 🔒)

---

## 섹션 2: 퀴즈 시스템

### 변경 내용
- 현재: 14문제 통합, 3회 제한, 카테고리 미분리
- 변경: 카테고리별 독립 퀴즈

### 퀴즈 규칙
| 항목 | 내용 |
|------|------|
| 형식 | 객관식 4지선다 (전 문제) |
| 문제 수 | 카테고리당 5~8문제 |
| 통과 기준 | 70% 이상 정답 |
| 재도전 | 횟수 제한 없음 |
| 문제 방향 | 지엽적 내용 제거, 실생활에서 중요한 내용 위주 |

### 뱃지 목록 (카테고리 기준, 노션 카테고리 확정 후 업데이트)
| 카테고리 ID | 뱃지 이름(KO) | 뱃지 이름(EN) |
|-------------|--------------|--------------|
| COURSE_REG  | 수강신청 박사 | Course Reg Expert |
| TRANSPORT   | 교통 박사     | Transport Expert  |
| FOOD        | 맛집 박사     | Food Expert       |
| CAMPUS_SITE | 사이트 박사   | Campus Site Expert |
| _(추가 예정)_ | _(노션 기준)_ | _(노션 기준)_ |

## 섹션 3: 백엔드 — 뱃지 API

### DB 변경
기존 테이블 변경 없음. 테이블 1개 추가:

```sql
CREATE TABLE member_badges (
  id         BIGSERIAL PRIMARY KEY,
  member_id  BIGINT NOT NULL REFERENCES members(id),
  badge_id   VARCHAR(50) NOT NULL,  -- ex: 'COURSE_REG'
  earned_at  TIMESTAMP NOT NULL DEFAULT NOW(),
  UNIQUE (member_id, badge_id)
);
```

뱃지 종류는 코드에 Enum으로 고정 (별도 테이블 불필요).

### API

```
POST /api/badges/{badgeId}
  - 인증 필요 (JWT)
  - 중복 획득 시 무시 (UNIQUE 제약)
  - Response: 200 OK

GET /api/members/{memberId}/badges
  - 공개 API (인증 불필요)
  - Response: [ { badgeId, badgeName, earnedAt }, ... ]
```

커뮤니티 팀원은 `GET /api/members/{memberId}/badges`만 호출하면 됩니다.

## 섹션 4: 프로필 화면

### 변경 내용
- 경희온도 완전 제거
- 뱃지 컬렉션 섹션으로 대체

### 뱃지 컬렉션 UI
```
[내 뱃지]
  🏅 수강신청 박사   🏅 맛집 박사
  🔒 교통 박사       🔒 사이트 박사
```
- 획득한 뱃지: 컬러 + 이름 표시
- 미획득 뱃지: 흑백 + 🔒 표시 → 도감 채우기 심리 유도

---

## 섹션 5: 커뮤니티 연동 (팀원 담당)

우리 팀이 제공하는 것:
- `GET /api/members/{memberId}/badges` API
- 응답 형태: `[{ badgeId, badgeName, earnedAt }]`

커뮤니티 팀원이 할 것:
- 게시글/답변 작성자 정보 조회 시 위 API 호출
- 보유 뱃지 중 대표 뱃지 1개(가장 최근 획득)를 이름 옆에 표시

- 뱃지 등급 시스템 (브론즈/실버/골드 등)

## 구현 순서

1. 노션 콘텐츠 → `khuGuide.ts` 교체 + 카테고리 확정
2. 백엔드: `member_badges` 테이블 + API 2개
3. 프론트: 카테고리별 퀴즈 분리 + 뱃지 획득 로직
4. 프론트: 프로필 화면 (경희온도 제거 + 뱃지 컬렉션)
5. 프론트: 가이드 카드에 뱃지 획득 여부 표시
